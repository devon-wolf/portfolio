Escape GIM

Site | Frontend | Backend

Live multiplayer escape game disguised as a simple corporate website

Escape GIM uses a React frontend with Socket.io, Express, and Node on the backend. It was developed over about 7 days by a team of 5 engineers, all working remotely.

Our concept was "a boring website, but it's a multiplayer escape room." Players are divided by the socket logic into rooms of three or fewer - rooms "lock" once any player has received a point, to prevent players entering mid-game. All player cursors appear on all screens in real-time, and effects one player has on the site affect other players as well. This was achieved through extensive use of sockets, React context providers, custom React hooks, and more CSS than any of us bargained for.

The parts of this project that I am most responsible for are the chat and the logic for the intermittent, game-state-dependent popups that advance the story. Both make use of sockets and custom hooks, and the popup logic uses our global game state context provider. We mob- and pair-programmed a good deal of this project, and managed our individual tasks via a ticketing system.

Deployed on Heroku and Netlify

Shame-b0t

GitHub

Snarky Discord bot for time and attention management

Shame-b0t has a special place in my heart - it was born from the writing sprint proctor project I had attempted that spurred me to get more serious about engineering. Based on user commands, the bot can either call out a user publicly for chatting during their designated focus time, or lock the user out of public channels in a Discord server completely for that duration.

During a 5 day sprint, our remote team of 4 used the Discord.js library in a Node application to build the bot. We used a continuously updating global state object to keep track of active users, their restrictions, and the time at which they should no longer be tracked.

My main contributions were leading the exploration/learning phase for my teammates and writing the bulk of the permissions management code, though we mob-programmed extensively throughout this project. We managed our tasks with a ticketing system and a focus on user stories.

Deployed on Heroku

Else by Elsewhere

Site | Frontend | Backend

Location guessing game using Google Maps and GeoDataSource

Else by Elsewhere is a full-stack React app that presents players with an image from the Google Maps Static Street View API, along with optional hints from the GeoDataSource API, and asks players to identify where in the world that image might be. Location data is stored in a PostgreSQL database and served by Node/Express. We worked in a remote team of 4 for a 5 day sprint, mob-programming nearly everything and organizing our work in a ticketing system.

Deployed on Heroku and Netlify

Forgotten Logic

Sliding number puzzle

A Small Text Editor

Site | Frontend | Backend

Collaborative text editor written in TypeScript using the SlateJS library, React, and Socket.io (so far)

This is a testing ground for building a robust collaborative text editor - there are many problems with its behavior at present, and it is currently just one document living in a giant array on the server. This is my main ongoing project, with the aim of developing a collaborative editing platform that is writer-friendly, easily shared with editors, easily exported to common file formats, and not data-hungry.

Deployed on Heroku and Netlify

Minion Bot

Bot | Dashboard | Server

Obnoxious Discord bot delivering Minion memes in response to trigger words

I built this bot for a friend using the Discord.js library, initially hardcoded with a set of starter responses and later expanded to allow the bot admin to add and remove responses and triggers to a Node/Express/Postgres backend via a React frontend. The bot makes a fetch to the API when it first comes online, and can be manually triggered to make a fresh fetch as needed.

Deployed on Heroku and Netlify

Cactus Bot

GitHub

Discord bot using the Merriam-Webster API to deliver thesaurus results

This bot was initially intended to a prickly proctor for writing sprints, and was the original project that inspired Shame-b0t. It was the experimentation ground for a lot of my bot and Discord.js learning, and ultimately the main functionality that came out of it was this thesaurus command.

It uses Superagent to hit the Merriam-Webster API with a search for the user-provided argument, then shapes those results into a user-friendly embed.

Deployed on Heroku

Bot Template

Ongoing project building a clear, versatile template for new coders to build Discord bots

I began to build this template to provide a clear starting point for folks new to bots, keeping in mind the struggles I had when trying to get started.
